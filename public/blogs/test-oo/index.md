收藏像把东西搬进仓库，写作才是在林子里修路。前者让人感到拥有，后者才让我们下次真的能够抵达。

## 从“我看过”退回到“我能找到”

很长一段时间，我的笔记按工具分类：浏览器里是网页，聊天软件里是截图，手机备忘录里是临时念头。每一处都不算混乱，但它们彼此之间没有路。

后来我只保留三种入口：**正在做的事、持续关心的问题、已经完成的记录**。分类不再追求正确，而是回答一个更实际的问题：下次需要它时，我会从哪里出发？

```ts
type Note = {
  question: string;
  source?: string;
  nextStep: string;
};
```


一条笔记若没有 `nextStep`，通常只是一件被妥善保存的旧物。

## 链接应该表达关系

双向链接最容易变成另一种收藏按钮。我给链接加了一条很轻的规则：在它旁边写清楚关系。

- 这是一个反例；
- 它解释了前文没有说明的原因；
- 这两个观点在同一个约束下冲突；
- 下一次实践时，从这里继续。

这样做会变慢，却能让知识库从文件夹变成地图。地图不需要保存每一棵树，只需要告诉你道路怎样相接。

## 每周只维护一小段路

我不再安排“整理整个知识库”这种注定失败的任务。周末只处理最近使用过的五条笔记：补一句上下文，删除失效链接，把真正有用的片段写进正在推进的文章。

> 好的系统不是从不混乱，而是允许你用很小的成本重新找到方向。

知识库最终服务的不是过去，而是下一次思考。如果一条笔记能让我少绕一次远路，它就已经完成了工作。
